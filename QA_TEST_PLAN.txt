Started: 2025-03-17
Released: -

Features discovered in the codebase:

* Admin Product Management
  - Create, read, update, delete products
  - Workflow builder with drag-and-drop steps
  - Product list with pagination, search, activeOnly filter
  - Product logs (PRODUCT_CREATED, PRODUCT_UPDATED, PRODUCT_DELETED)
  - Soft delete (status DELETED, deleted_at)
  - completeCreate for first save after create (merge image/template keys without version bump)

* Product Versioning
  - base_id groups versions
  - New row per edit (old INACTIVE, new ACTIVE)
  - workflowDeepEqual prevents no-op version bump
  - Replaced S3 keys deleted on update
  - category_display_order, product_display_order from workflow config

* Issuer Application Workflow
  - Step keys: financing_type, financing_structure, contract_details, invoice_details, company_details, business_details, financial_statements, supporting_documents, declarations, review_and_submit
  - Create application with product_id in financing_type, product_version
  - PATCH step updates JSON columns, last_completed_step
  - Amendment mode: only flagged sections editable
  - forceRewindToStep for admin correction

* Admin Review Flow
  - Section review: approve, reject, request-amendment, reset-to-pending
  - Item review: approve, reject, request-amendment for invoices and documents
  - Pending amendments: add, update, remove, submit
  - Send contract offer, send invoice offer
  - Application status: UNDER_REVIEW, APPROVED, REJECTED
  - reopen-for-correction

* Amendment Flow
  - addPendingAmendment (draft), submitPendingAmendments (sent)
  - getAmendmentAllowedSections from REQUEST_AMENDMENT remarks
  - acknowledgeWorkflow before resubmit
  - resubmitApplication: clear remarks, create revision, increment review_cycle, RESUBMITTED

* Contract Financing
  - Create contract, link to application
  - contract_details, customer_details, offer_details
  - Document upload presigned URL
  - Status: DRAFT, SUBMITTED, OFFER_SENT, APPROVED, REJECTED, AMENDMENT_REQUESTED, WITHDRAWN
  - existing_contract link from financing_structure
  - invoice_only structure blocks contract_details updates

* Invoice Financing
  - Create invoice under application, optional contract_id
  - details: number, value, maturity_date, financing_ratio_percent, document
  - Document upload, S3 path under applications/
  - Status transitions aligned with contract
  - min/max financing ratio from product config
  - Maturity date validation: not past, within contract window

* AWS S3 Document Storage
  - Presigned upload URLs for products, applications, contracts, invoices
  - Key formats: products/{id}/v{n}-{date}-{cuid}.ext, applications/{id}/..., contracts/{id}/..., invoices use applications path
  - Version bump on replace, old object deleted
  - Product soft delete: S3 NOT deleted
  - Max 5MB, PDF for docs, PNG for product image

* Application Step Resume Logic
  - last_completed_step source of truth
  - allowedMaxStep = lastCompletedStep + 1
  - URL ?step=N validated and redirected
  - wizardState synced from application on load

* Application Version Checking
  - New application: revalidate product before create
  - VersionMismatchModal: PRODUCT_DELETED, PRODUCT_INACTIVE, PRODUCT_VERSION_CHANGED
  - useProductVersionGuard on edit page

* Organization Integration
  - issuer_organization, corporate_onboarding_data
  - RegTank refresh-corporate-entities, refresh-aml-status, refresh-corporate-status
  - director_aml_status, director_kyc_status, business_aml_status
